# Data Manifest

This repository keeps lightweight metadata and split definitions in the public
tree. Raw inputs and generated outputs are ignored by default so readers can
reproduce them locally or retrieve large artifacts from an external release.

## Required Inputs

| Artifact | Path | Public Git status | Notes |
| --- | --- | --- | --- |
| Raw MP table | `data/raw/mp_total.csv` | ignored | Source table used to derive labels and chemistry metadata. |
| CIF structures | `data/raw/cifs/*.cif` | ignored | Structure inputs for graph features, RDF preprocessing, and pretrained ALIGNN descriptors. |
| Atom initialization | `data/raw/atom_init.json` | ignored | CGCNN-style atom initializer. |
| Processed multimodal cache | `data/processed/publication/` | tensor files ignored | Contains aligned tabular/RDF/structure tensors. The CSV/JSON manifests can be tracked; `*.pt` feature tensors should be externally hosted if needed. |
| Sample order | `data/sample_order/sample_order_keep_last.csv` | retained | One row per retained `id_discharge`; used only to keep split generation reproducible. |
| Random splits | `data/splits/publication/{target}/seed_{seed}/` | retained | Target-specific random train/val/test IDs for seeds 0-4. |
| OOD splits | `data/splits/publication_ood/` | generated | Composition-cluster and working-ion holdout splits generated by `imfuse split-ood`. |

## Generated Outputs

| Artifact | Path | Notes |
| --- | --- | --- |
| Final metrics/configs | `results/final_publication/` | Generated by the publication runner. |
| OOD metrics/configs | `results/final_publication_ood/` | Generated by `scripts/run_ood_publication.sh`. |
| Predictions | `results/predictions/final_publication_*` | Generated prediction CSVs used for subgroup, OOD, and robustness summaries. |
| Explanation metrics | `results/explanation_validation/` | Generated permutation, deletion, and atom-ablation audit outputs. |
| Publication figures | `figures/final_publication/` | Generated figures for random-split, baseline, dropout, and subgroup analyses. |
| Explanation figures | `figures/explanation_validation/` | Generated feature-importance and deletion-curve audit figures. |

These folders are not committed to Git. They are produced by the commands in
`docs/reproduce.md` and `docs/reproducibility.md`.

## Large Artifacts Not Intended For Git

The following are ignored by `.gitignore` to keep the public repository usable:

- `data/raw/`
- `data/processed/**/*.pt`
- `results/`
- `figures/`
- `results/**/checkpoints/`
- `results/**/models/`
- `results/**/embeddings/`
- `*.pt`, `*.pkl`, `*.npy`
- local environments, logs, and `_local_archive/`

For full bitwise reruns, publish these large inputs/checkpoints through an external data release and point readers to it from the README.

## Rebuilding The Data Foundation

Use `imfuse prepare-data` to rebuild the public data foundation from an external
`mp_total.csv` and CIF source:

```bash
imfuse prepare-data \
  --target-col average_voltage \
  --mp-total-url "$MP_TOTAL_URL" \
  --cif-archive-url "$CIF_ARCHIVE_URL" \
  --seeds 0 1 2 3 4 \
  --preprocess
```

The command writes target-specific labels, CIF coverage, split manifests, and
optional modality caches under `data/`. See `docs/data_preparation.md` for the
full local/URL/Materials Project options.
